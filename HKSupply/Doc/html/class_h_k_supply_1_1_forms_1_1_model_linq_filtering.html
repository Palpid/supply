Con los números con decimales tengo un "problema", si los parseo desde un string el decimal lo marca el caracter "," pero en el teclado numérico es el "." Al parsear tengo que cambiar el punt (".") por coma(","), pero después para que lo coja el linq tiene que ser punto ("."), así que hay que hacer el juego de reemplazar dos veces mientras no encuentre algo mejor.

El separador decimal varía entre punto (".") y coma(",") según la cultura. Para trabajar con ello usaremos InvariantCulture
